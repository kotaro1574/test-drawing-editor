## プロジェクト概要

Fabric.js ベースのお絵描き Web アプリで、ユーザーが描いた絵を**サービス専用 X アカウント**に画像付きツイートとして投稿し、投稿した画像の**一覧ギャラリーページ**を作成する。

### 想定フロー

1. ユーザーが Canvas でお絵描き
2. 「投稿」ボタンで画像を X に投稿（サービス専用アカウント経由）
3. 投稿時に tweet ID + 画像 URL を DB に保存
4. ギャラリーページで DB の画像 URL を使って一覧表示

## 画像投稿フローと料金シミュレーション

### API 呼び出しフロー

```
1. v1.uploadMedia(buffer)    → media_id 取得（課金対象外※）
2. v2.tweet({ text, media }) → tweet_id 取得（コンテンツ: 作成 = $0.010）
3. GET /2/tweets/:id         → 画像URL 取得（投稿: 読み取り = $0.005）
   ※ expansions=attachments.media_keys&media.fields=url を指定
4. DB に tweet_id + 画像URL を保存
```

※ メディアアップロードの課金について: 料金表に「メディア: 作成」の項目が存在しないため、アップロード自体はコンテンツ作成に含まれると推定。

### 重要: ツイート作成レスポンスに画像URLは含まれない

`POST /2/tweets` のレスポンスは以下の形式のみ:

```json
{
  "data": {
    "id": "1234567890",
    "text": "絵を描いたよ！"
  }
}
```

画像 URL（`pbs.twimg.com/media/...`）を取得するには、**追加の GET リクエスト**が必要。

### 1回の投稿にかかるコスト

| ステップ             | API 呼び出し        | コスト            |
| -------------------- | ------------------- | ----------------- |
| メディアアップロード | `v1.uploadMedia()`  | $0（推定）        |
| ツイート投稿         | `POST /2/tweets`    | **$0.010**        |
| 画像 URL 取得        | `GET /2/tweets/:id` | **$0.005**        |
| **合計**             |                     | **$0.015 / 投稿** |

### 月間コストシミュレーション

| 投稿数/月      | X API コスト | 日本円（$1=160円） |
| -------------- | ------------ | ------------------ |
| 10回           | $0.15        | 約24円             |
| 30回（1日1回） | $0.45        | 約72円             |
| 100回          | $1.50        | 約240円            |
| 300回          | $4.50        | 約720円            |
| 1,000回        | $15.00       | 約2,400円          |

**ギャラリーページの表示コスト: $0**
DB に保存した画像 URL を `<img>` タグで直接表示するため、X API は呼ばない。

**$5 のクレジットで約333回投稿可能。**

## pbs.twimg.com 画像URLの永続性

### X を「画像ストレージ」として使う場合のリスク

| リスク                     | 深刻度    | 詳細                                                                                                                            |
| -------------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------- |
| ツイート削除で画像消滅     | 🔴 高     | ツイートが削除されると画像 URL もアクセス不可になる可能性。公式の永続性保証なし                                                 |
| URL フォーマット変更       | 🟡 中     | X 社は過去に `:large` サフィックス方式から `?format=jpg&name=large` クエリパラメータ方式に変更した実績あり                      |
| ブラウザのトラッキング保護 | 🟡 中     | Firefox は `pbs.twimg.com` を「ソーシャルメディアトラッカー」として分類。トラッキング保護が有効なユーザーには画像が表示されない |
| ホットリンク制限           | 🟡 中     | 現時点では外部サイトからの埋め込みは可能だが、X 社がいつでもポリシーを変更できる                                                |
| 画像の自動変換             | 🟠 低〜中 | PNG でアップロードしても JPEG に変換される場合がある。お絵描きアプリとしては画質劣化のリスク                                    |
| CORS の問題                | 🟢 低     | 現時点では `<img>` タグでの表示は問題なし。JavaScript での fetch には制限あり                                                   |

### 公式保証について

- X 社は画像 URL の永続性について**公式ドキュメントで一切保証していない**
- 画像 URL は X 社の内部インフラ（Photo Blobstore）に依存しており、仕様変更の可能性あり

## Cloudflare R2 による画像ストレージ代替案

### Cloudflare R2 とは

AWS S3 互換のオブジェクトストレージサービス。最大の特徴は**エグレス料金（データ転送料）が完全無料**であること。

### Free Tier（無料枠）

| 項目                         | 無料枠             | 備考                            |
| ---------------------------- | ------------------ | ------------------------------- |
| **ストレージ**               | **10 GB/月**       | Standard ストレージのみ         |
| **Class A 操作（書き込み）** | **100万回/月**     | PUT, POST, COPY, LIST など      |
| **Class B 操作（読み取り）** | **1,000万回/月**   | GET, HEAD など                  |
| **エグレス（データ転送）**   | **$0（完全無料）** | どれだけ転送しても無料          |
| **期間制限**                 | **なし（永続）**   | AWS S3 と違い、期間限定ではない |

### 有料枠（Free Tier 超過時）

| 項目         | 単価            |
| ------------ | --------------- |
| ストレージ   | $0.015 / GB-月  |
| Class A 操作 | $4.50 / 100万回 |
| Class B 操作 | $0.36 / 100万回 |
| エグレス     | **$0（無料）**  |

### お絵描きアプリでの使用量見積もり

前提: 画像サイズ 平均 500KB（1000×1000px PNG）

| 投稿数/月 | ストレージ累計（6ヶ月後） | Class A  | Class B（表示） | 月額コスト         |
| --------- | ------------------------- | -------- | --------------- | ------------------ |
| 30回      | 90MB                      | 30回     | 〜1,000回       | **$0（無料枠内）** |
| 100回     | 300MB                     | 100回    | 〜5,000回       | **$0（無料枠内）** |
| 1,000回   | 3GB                       | 1,000回  | 〜50,000回      | **$0（無料枠内）** |
| 10,000回  | 30GB                      | 10,000回 | 〜500,000回     | 約$0.30/月         |

**10GB = 約 20,000 枚の画像を保存可能（500KB/枚想定）。Free Tier だけで長期間運用できる。**

### R2 の利点

- **S3 互換 API**: `@aws-sdk/client-s3` がそのまま使える
- **エグレス無料**: ギャラリーページで大量アクセスがあっても転送料 $0
- **公開バケット**: カスタムドメインまたは `r2.dev` サブドメインで直接画像配信可能
- **CORS 完全対応**: 外部サイトからの読み込みも問題なし
- **画像そのまま保存**: PNG は PNG のまま。変換・圧縮されない
- **Presigned URL 対応**: 一時的なアップロード/ダウンロード URL の発行可能
- **永続性**: 自分で削除しない限り、データは保持される

### R2 の注意点

- Cloudflare アカウントの作成が必要（無料）
- カスタムドメインで配信する場合、Cloudflare DNS を使う必要あり
- `r2.dev` サブドメインは開発用途向け（本番では制限あり）
- Next.js の API Route からアップロードする場合、`@aws-sdk/client-s3` のインストールが必要

## 推奨アーキテクチャ

### 案 A: X API のみ（シンプル）

```
Canvas → API Route → X API (upload + tweet) → GET tweet → DB に URL 保存
ギャラリー → DB から URL 取得 → <img src="pbs.twimg.com/...">
```

- ✅ シンプル
- ✅ SNS 拡散あり
- ⚠️ 画像永続性リスクあり
- ⚠️ Firefox 表示問題

### 案 B: Cloudflare R2 のみ（堅実）

```
Canvas → API Route → R2 に画像保存 → DB に R2 URL 保存
ギャラリー → DB から URL 取得 → <img src="r2-domain/...">
```

- ✅ 画像永続性保証
- ✅ 完全無料（Free Tier 内）
- ✅ 全ブラウザ対応
- ❌ SNS 拡散なし

### 案 C: X API + Cloudflare R2 併用（推奨）

```
Canvas → API Route → R2 に画像保存 + X API で投稿 → DB に R2 URL + tweet_id 保存
ギャラリー → DB から R2 URL 取得 → <img src="r2-domain/...">
X 上 → ツイートとして画像が残り、SNS 拡散効果あり
```

- ✅ 画像永続性保証（R2）
- ✅ SNS 拡散あり（X）
- ✅ 全ブラウザ対応（R2 URL で表示）
- ✅ X の投稿が削除されてもギャラリーに影響なし
- ✅ 画像 URL 取得の GET が不要（R2 URL を使うため、$0.005/回 節約）
- ⚠️ 実装がやや複雑（2サービス連携）

## 次のステップ

### ユーザーアクション（必要）

1. **X Developer Portal** でアカウント作成 & API キー取得
   - Consumer Key / Consumer Secret
   - Access Token / Access Token Secret
2. クレジット購入（$5〜）& 支出上限設定
3. **Cloudflare アカウント**作成（R2 を使う場合）
   - R2 バケット作成
   - API トークン取得

### 実装タスク（API キー取得後）

1. `twitter-api-v2` パッケージインストール
2. `@aws-sdk/client-s3` パッケージインストール（R2 使用時）
3. `/api/post-to-x/route.ts` API Route 作成
4. `.env.local` に API キー設定
5. `DrawingCanvas.tsx` の Web Share API 実装を差し替え
6. ギャラリーページ作成
